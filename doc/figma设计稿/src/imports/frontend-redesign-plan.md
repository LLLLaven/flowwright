# FlowWright 前端重设计方案

> 版本：v1.0 · 日期：2026-07-03
> 当前状态：Phase 1 基础功能已实现，UI 设计需要全面升级

---

## 一、现状分析

### 当前问题
1. **视觉设计过于简陋**：基础的灰白色调，缺乏品牌识别度
2. **信息层次不清晰**：工具栏、画布、配置面板融为一体，缺乏视觉分隔
3. **交互体验欠佳**：
   - 节点样式单调，状态区分不够明显
   - 缺乏操作反馈和过渡动画
   - 工具栏功能堆叠在一起，缺乏分组
4. **专业感不足**：不像一个专业的开发工具，更像原型 demo
5. **缺失关键功能 UI**：
   - 没有全局状态栏显示运行信息
   - 缺少快捷操作面板
   - 没有实时日志查看器
   - 缺少工作流模板库

### 优势保留
- React Flow 的基础交互是好的（拖拽、连线）
- 节点配置面板的结构是合理的
- Monaco Editor 集成是正确的选择

---

## 二、设计目标

### 视觉定位
**现代化专业开发工具** — 对标 VS Code、Linear、Figma 的 UI 品质

### 设计原则
1. **清晰的信息层次**：主工作区、侧边栏、面板层次分明
2. **高效的操作流**：常用功能一键触达，减少点击次数
3. **优雅的反馈**：每个操作都有视觉/动画反馈
4. **专业的细节**：图标、字体、间距、圆角统一规范

### 5.1 节点卡片（AgentNode）

#### 视觉升级
```tsx
// 尺寸

圆角: 12px (原 8px → 更圆润)

  默认: 0 2px 8px rgba(0,0,0,0.1)
  选中: 0 4px 16px rgba(99,102,241,0.3) + 发光效果

// 结构
┌────────────────────────┐
│ [图标] 节点名称    [●] │  ← Header: 左图标 + 标题 + 右状态点
│ 🤖 Agent              │  ← Type badge
├────────────────────────┤
│ Provider: DeepSeek    │  ← 元信息（可折叠）
│ Model: v4-flash       │
├────────────────────────┤  ← 仅 running 时显示
│ ▸ Generating code...  │  ← 流式预览（滚动显示最后 80 字符）
└────────────────────────┘
```

#### 状态指示器
- **Idle**: 灰色虚线边框 + 灰色状态点
- **Running**: 蓝色实线边框 + 蓝色脉动状态点 + 流式预览区
- **Completed**: 绿色边框 + 绿色勾选状态点
- **Rejected**: 红色边框 + 红色叉号状态点 + 错误提示
- **Awaiting Human**: 琥珀色边框 + 琥珀色暂停状态点 + "等待审核"标签

#### 交互细节
- **悬停**: 显示快捷操作按钮（编辑、复制、删除）
- **选中**: 发光边框 + 周围其他节点半透明
- **拖拽**: 拖拽时节点略微放大 + 拖影效果
- **连线**: 连接柄（Handle）仅在悬停时显示，默认隐藏

## 六、动画与过渡

### 微交互动画
```css
/* 节点选中 */
@keyframes node-select {
  0% { transform: scale(1); box-shadow: 0 2px 8px rgba(0,0,0,0.1); }
  100% { transform: scale(1.02); box-shadow: 0 4px 16px rgba(99,102,241,0.3); }
}

/* 状态点脉动 */
@keyframes status-pulse {
  0%, 100% { opacity: 1; transform: scale(1); }
  50% { opacity: 0.5; transform: scale(1.2); }
}

/* 面板滑入 */
@keyframes panel-slide-in {
  from { transform: translateX(-100%); opacity: 0; }
  to { transform: translateX(0); opacity: 1; }
}

/* 按钮按下 */
button:active {
  transform: scale(0.96);
  transition: transform 0.1s;
}
```

### 过渡时机
- **节点状态变化**: 0.3s ease
- **面板打开/关闭**: 0.25s cubic-bezier(0.4, 0, 0.2, 1)
- **按钮悬停**: 0.15s ease
- **工作流执行**: 边的动画 + 节点状态点脉动

## 十一、技术选型补充

### UI 组件库（推荐）
- **Radix UI**: 无样式的高质量组件原语（对话框、下拉菜单等）
- **Headless UI**: Tailwind 官方，轻量
- **或保持当前无依赖，手写组件**（更轻量，但开发时间长）

### CSS 方案
- **当前**: 内联样式 → 建议迁移到 CSS Modules 或 Tailwind CSS
- **Tailwind CSS**: 快速开发，类名直观
- **CSS Modules**: 作用域隔离，无冲突

### 动画库（可选）
- **Framer Motion**: React 动画库，声明式
- **React Spring**: 基于物理的动画

## 十四、性能优化

### React 优化
- 使用 `React.memo` 包裹所有节点组件
- 使用 `useCallback` 和 `useMemo` 缓存函数和对象
- 虚拟滚动（日志查看器，节点数 > 100）

### 渲染优化
- React Flow 的 `nodesDraggable`、`nodesConnectable` 等属性按需开启
- 流式文本更新节流（100ms）
- 大型工作流（>50 节点）时隐藏 MiniMap

---

## 总结

这个重设计方案将 FlowWright 从"原型 demo"提升到"专业开发工具"水准。核心改进：

1. **视觉**: 深色主题 + 紫蓝渐变品牌色 + 现代 UI
2. **布局**: 清晰的六区布局，信息层次分明
3. **交互**: 动画反馈 + 拖拽添加 + 快捷键
4. **功能**: 节点库 + 日志查看器 + 实时状态栏

实施优先级：Phase 1 > Phase 2 > Phase 3。Phase 4 可在 MVP 验证后迭代。
